Masalahnya

Di era digital seperti sekarang, banyak warung makan, kantin, dan restoran kecil masih mengandalkan cara konvensional dalam menerima pesanan makanan, yaitu melalui telepon, pesan WhatsApp, atau bahkan datang langsung ke tempat. Cara ini menimbulkan berbagai permasalahan nyata yang sering terjadi sehari-hari, antara lain:


a. Kesalahan Pencatatan Pesanan

Ketika pesanan diterima melalui chat atau telepon, pemilik toko harus mencatat secara manual. Hal ini sangat rentan terhadap kesalahan, misalnya salah mencatat nama menu, jumlah porsi, atau alamat pengiriman pelanggan.

B.Tidak Ada Informasi Stok Real-time

Pemilik tidak mengetahui secara langsung menu mana yang sudah habis terjual. Akibatnya, pelanggan sudah memesan tetapi baru diberitahu menu habis setelah menunggu lama.

c. Sulit Melacak Riwayat Pesanan

Karena semua dicatat manual, sulit bagi pemilik untuk mengetahui berapa total pesanan seorang pelanggan, menu apa yang paling laris, atau berapa pendapatan hari ini.

d. Pelayanan Lambat dan Tidak Efisien

Proses pemesanan yang panjang dan manual membuat pelanggan harus menunggu konfirmasi lebih lama, sehingga menurunkan kepuasan pelanggan.

Kesimpulan Masalah:

Dibutuhkan sebuah Sistem Pemesanan Makanan Online berbasis OOP yang mampu mengelola data menu makanan, data pelanggan, dan transaksi pesanan secara otomatis, terstruktur, dan efisien.

Kenapa Masalah Ini Perlu Diselesaikan dengan OOP?

Masalah sistem pemesanan makanan online sangat tepat diselesaikan menggunakan pendekatan Object-Oriented Programming (OOP) karena beberapa alasan berikut:


a. Ada Banyak Entitas Nyata yang Bisa Dimodelkan sebagai Class

Dalam sistem ini, terdapat entitas-entitas nyata yang masing-masing memiliki data (atribut) dan perilaku (method) tersendiri:

- MenuMakanan → memiliki nama, harga, kategori, dan status ketersediaan

- Pelanggan → memiliki identitas dan bisa melakukan tindakan pemesanan

- Pesanan → menghubungkan pelanggan dengan menu yang dipesan, menyimpan status transaksi


b. Enkapsulasi Melindungi Data Penting

Dengan OOP, data sensitif seperti status ketersediaan menu tidak bisa diubah sembarangan dari luar class. Perubahan hanya bisa dilakukan melalui method khusus seperti tandaiHabis() dan tandaiTersedia(). Ini mencegah bug dan kesalahan data yang tidak disengaja.


c. Abstraksi Menyembunyikan Kompleksitas

Ketika pelanggan memesan, cukup memanggil buatPesanan(pesanan). Di balik method tersebut, sistem otomatis mengecek ketersediaan menu, memproses pesanan, dan memperbarui data — tanpa perlu diketahui detailnya oleh pihak luar.


d. Interaksi Antar Objek Mencerminkan Dunia Nyata

Sama seperti di dunia nyata, dalam sistem ini objek-objek saling berinteraksi:

- Objek Pelanggan memanggil method pada objek Pesanan

- Objek Pesanan memanggil method pada objek MenuMakanan

Ini adalah gambaran nyata bagaimana OOP bekerja dalam konteks kehidupan sehari-hari.


e. Mudah Dikembangkan (Extensible)

Jika suatu saat ingin menambahkan fitur baru seperti Kurir, Rating Makanan, atau Metode Pembayaran, cukup menambahkan class baru tanpa mengubah class yang sudah ada. Ini tidak bisa dilakukan semudah ini jika menggunakan pendekatan prosedural.

Langkah Penyelesaian Masalah

1. Mengidentifikasi objek utama, yaitu MenuMakanan, Pelanggan, dan Pesanan.

2. Membuat class beserta atribut dan methodnya:

MenuMakanan: nama, harga, kategori, tersedia.

Pelanggan: nama, ID pelanggan, alamat, total pesanan.

Pesanan: ID pesanan, menu, pelanggan, jumlah, total harga, status.

3. Mengimplementasikan program, yaitu pelanggan melakukan pemesanan, sistem mengecek ketersediaan menu, memproses pesanan, dan menampilkan hasil transaksi.

4. Menerapkan konsep OOP:

Class & Object untuk memodelkan data.

Enkapsulasi untuk melindungi data.

Abstraksi untuk menyederhanakan proses.

Interaksi Objek untuk menjalankan transaksi.

Constructor untuk memberikan nilai awal objek.


Hasil: Sistem pemesanan makanan online dapat mengelola menu, pelanggan, dan pesanan secara lebih terstruktur, cepat, dan efisien.

  class MenuMakanan {
  String nama;
  double harga;
  String kategori;
  bool tersedia;

  MenuMakanan(this.nama, this.harga, this.kategori, this.tersedia);

  void tampilkanInfo() {
    print("Nama     : $nama");
    print("Harga    : Rp$harga");
    print("Kategori : $kategori");
    print("Status   : ${tersedia ? 'Tersedia' : 'Habis'}");
  }

  void tandaiHabis() {
    if (tersedia) {
      tersedia = false;
      print("Menu '$nama' sekarang habis.");
    } else {
      print("Menu '$nama' memang sudah habis.");
    }
  }

  void tandaiTersedia() {
    if (!tersedia) {
      tersedia = true;
      print("Menu '$nama' sekarang tersedia kembali.");
    } else {
      print("Menu '$nama' memang sudah tersedia.");
    }
  }
}

// CLASS PELANGGAN
// ================================
class Pelanggan {
  String nama;
  String idPelanggan;
  String alamat;
  int totalPesanan;

  Pelanggan(this.nama, this.idPelanggan, this.alamat)
      : totalPesanan = 0;

  void tampilkanInfo() {
    print("Nama Pelanggan : $nama");
    print("ID Pelanggan   : $idPelanggan");
    print("Alamat         : $alamat");
    print("Total Pesanan  : $totalPesanan kali");
  }

  void buatPesanan(Pesanan pesanan) {
    pesanan.prosesPesanan();
    if (pesanan.status == "Berhasil") {
      totalPesanan++;
    }
  }
}

// ================================
// CLASS PESANAN
// ================================
class Pesanan {
  String idPesanan;
  Pelanggan pelanggan;
  MenuMakanan menu;
  int jumlah;
  double totalHarga;
  String status;

  Pesanan(this.idPesanan, this.pelanggan, this.menu, this.jumlah)
      : totalHarga = menu.harga * jumlah,
        status = "Menunggu";

  void prosesPesanan() {
    if (menu.tersedia) {
      status = "Berhasil";
      menu.tandaiHabis();
      print("Pesanan '$idPesanan' berhasil dibuat!");
    } else {
      status = "Gagal";
      print("Pesanan gagal, menu '${menu.nama}' sedang habis.");
    }
  }

  void tampilkanStruk() {
    print("====== STRUK PESANAN ======");
    print("ID Pesanan  : $idPesanan");
    print("Pelanggan   : ${pelanggan.nama}");
    print("Alamat      : ${pelanggan.alamat}");
    print("Menu        : ${menu.nama}");
    print("Jumlah      : $jumlah porsi");
    print("Total Bayar : Rp$totalHarga");
    print("Status      : $status");
    print("===========================");
  }
}

void main() {
  // Membuat objek menu makanan
  MenuMakanan m1 = MenuMakanan("Nasi Goreng Spesial", 25000, "Makanan", true);
  MenuMakanan m2 = MenuMakanan("Es Teh Manis", 8000, "Minuman", true);
  MenuMakanan m3 = MenuMakanan("Mie Ayam Bakso", 20000, "Makanan", false);

  // Membuat objek pelanggan
  Pelanggan p1 = Pelanggan("Rina", "P001", "Jl. Melati No. 5");
  Pelanggan p2 = Pelanggan("Dika", "P002", "Jl. Kenanga No. 12");

  // ---- Informasi Awal ----
  print("--- Info Menu Makanan Awal ---");
  m1.tampilkanInfo();
  print("");
  m2.tampilkanInfo();
  print("");
  m3.tampilkanInfo();

  // ---- Proses Pemesanan ----
  print("\n--- Rina Memesan Nasi Goreng ---");
  Pesanan ps1 = Pesanan("ORD001", p1, m1, 2);
  p1.buatPesanan(ps1);

  print("\n--- Dika Memesan Menu yang Habis ---");
  Pesanan ps2 = Pesanan("ORD002", p2, m3, 1);
  p2.buatPesanan(ps2);

  print("\n--- Dika Memesan Es Teh Manis ---");
  Pesanan ps3 = Pesanan("ORD003", p2, m2, 3);
  p2.buatPesanan(ps3);

  // ---- Struk Pesanan ----
  print("\n--- Struk Pesanan Berhasil ---");
  ps1.tampilkanStruk();
  print("");
  ps3.tampilkanStruk();

  // ---- Info Akhir ----
  print("\n--- Info Menu Setelah Transaksi ---");
  m1.tampilkanInfo();

  print("\n--- Info Pelanggan Setelah Pesan ---");
  p1.tampilkanInfo();
  print("");
  p2.tampilkanInfo();
}

kesimpulan

- Sistem Pemesanan Makanan Online adalah contoh masalah nyata yang sangat tepat diselesaikan menggunakan OOP, karena:

- Setiap entitas dalam sistem bisa dimodelkan sebagai class dengan atribut dan method yang jelas, yaitu MenuMakanan, Pelanggan, dan Pesanan

- Konsep enkapsulasi menjaga data agar tidak diubah sembarangan. Status menu hanya bisa diubah lewat method yang sudah ditentukan.

- Konsep abstraksi membuat kode lebih sederhana di sisi pengguna. Cukup panggil satu method, sistem menangani prosesnya sendiri.

- Interaksi antar objek dalam sistem ini mencerminkan bagaimana dunia nyata bekerja, di mana pelanggan berinteraksi dengan pesanan, dan pesanan berinteraksi dengan menu makanan.


Dengan menggunakan OOP, kode menjadi lebih terstruktur, mudah dipahami, dan mudah dikembangkan di masa mendatang.
